## Data schema (`~/.job-tracker/data.json`)

```
{
  jobs: [{ id, title, company, location, url, description, status, notes, kit, created_at, updated_at }],
  settings: { profile (string), openrouter_api_key, openrouter_model, apify_api_key, apify_actor_id, keyword_model },
  recommendations: [ RecommendationItem ],
  nextId: number,
  nextRecId: number
}
```

`kit` shape: `{ resume_content, resume_latex, cover_letter, interview_questions, company_brief, ats_analysis }`  
`ats_analysis` shape: `{ score (0–100 | null), matched_keywords[], missing_keywords[], keywords_extracted[] }`  
`resume_content` shape: `{ coursework, technical_skills, business_skills, experiences[], projects[] }`  
`experiences[]`: `{ company, location, title, dates, bullets[] }`  
`projects[]`: `{ title, tools, date, bullets[] }`

`RecommendationItem`: `{ id, title, company, location, url, description, match_reasoning, source ("apify"|"ai"), created_at }`

`keyword_model` setting: model used for ATS keyword extraction — defaults to `openai/gpt-4o-mini` (fast/cheap); override via `PUT /api/settings`.

## LaTeX resume — current spacing state (as of Jun 10)

Template style: Jake Gutierrez (fullpage/titlesec/fancyhdr). Generated by `buildResumeLatex(c)` in `server.js`.

**Current macro values:**
```latex
\setlength{\parskip}{0pt}                          % kills mystery gaps between bullets
\titleformat{\section}{\vspace{-4pt}...}{}{}{}[\titlerule \vspace{-4pt}]
\resumeSubheading:       \vspace{-1pt}\item ... \end{tabular*}\vspace{-5pt}
\resumeProjectHeading:   \item ... \end{tabular*}\vspace{-5pt}
\resumeSubHeadingListEnd: \end{itemize}\vspace{0pt}
\resumeItemListStart:    \begin{itemize}[itemsep=2pt, topsep=2pt, parsep=0pt, partopsep=0pt]
\resumeItemListEnd:      \end{itemize}\vspace{-2pt}
\resumeItem:             \item\small{#1}            % NO \vspace{-2pt} — caused inconsistent gaps
```

**Key lessons learned (do not revert these):**
- `\vspace{-2pt}` inside `\resumeItem` caused large random gaps after single-line bullets — removed permanently
- `\setlength{\parskip}{0pt}` is required to prevent paragraph spacing from leaking into list gaps
- `itemsep=0pt` looks cramped; `itemsep=2pt` with `topsep=2pt` gives natural breathing room

**If resume spills to 2 pages:** tighten in this order:
1. `\resumeSubHeadingListEnd` vspace: `0pt` → `-2pt`
2. `\resumeItemListEnd` vspace: `-2pt` → `-4pt`
3. `topsep` in `\resumeItemListStart`: `2pt` → `0pt`
4. Section vspace: `-4pt` → `-6pt` (before and after rule)

**If resume has too much blank at bottom:** loosen in this order:
1. `topsep` in `\resumeItemListStart`: `2pt` → `4pt`
2. `\resumeSubHeadingListEnd` vspace: `0pt` → `2pt`
3. Reduce `\resumeItem` count check — AI may be generating too few bullets

## AI kit prompt rules (enforced in `buildKitPrompt`)

- Exactly **3 bullets per experience** — never fewer
- **2–3 bullets per project**, **3–4 projects** total
- Use `**bold**` around key metrics (converted to `\textbf{}` by `tex()`)
- All work experiences kept with original company/location/title/dates — only bullets are tailored
- Job title suffixes like "– Extern", "– Externship", "– Contract" are forbidden — stripped by `cleanTitle()` in code
- Bullets must be specific and quantified — forbidden endings: "supporting X", "enhancing Y", "showcasing skills in Z", "demonstrating skills relevant to Z", "providing insights into X", "across multiple scenarios" (without a number)
- Project bullets must state a real output: MAPE figure, valuation range, IRR, % change, etc.
- **ATS keywords** (extracted from JD before prompt is built) are injected into the prompt — AI is told to use them naturally across skills and bullets
- **Chain-of-thought** step: prompt instructs AI to identify 5 critical JD requirements and map each experience to 1–2 of them before writing bullets
- **Cover letter structure**: P1 company-specific hook (references company_brief), P2 strongest metric, P3 trajectory bridge, P4 CTA — generic openers ("I am excited to apply") are forbidden
- **Bullet self-check**: prompt instructs AI to verify each bullet has a strong action verb, a quantity, an ATS keyword, and no forbidden ending before finalizing

## Post-processing pipeline (`server.js`)

After AI returns JSON, before building LaTeX:

1. **`extractJdKeywords(jdText, api_key, keywordModel)`** — called first; extracts 10–15 ATS keywords from JD via `keyword_model`; result injected into `buildKitPrompt` and later used for scoring
2. **`sanitizeResumeContent(c)`** — strips "– Extern/Externship/Contract" etc. from titles via `cleanTitle()`, caps bullets at 3, strips internal newlines from bullet text (prevents LaTeX paragraph spacing leaks)
3. **Auto bullet-fill (batch)** — if any experience has < 3 bullets, a **single** API call via `batchFillBullets()` generates all missing bullets at once; re-sanitizes after fill
4. **`computeAtsScore(resumeContent, keywords)`** — pure JS; checks which extracted keywords appear in the final resume text; stores result in `kit.ats_analysis`
5. **`buildResumeLatex(c)`** — builds LaTeX from sanitized content

## API endpoints

| Method | Route | Purpose |
|--------|-------|---------|
| GET | `/api/jobs` | List all jobs |
| GET | `/api/jobs/:id` | Get single job |
| POST | `/api/jobs` | Create job |
| PUT | `/api/jobs/:id` | Update job fields |
| DELETE | `/api/jobs/:id` | Delete job |
| POST | `/api/jobs/:id/kit` | Generate AI kit (resume, cover letter, interview Q&A, ATS score) |
| GET | `/api/jobs/:id/kit/ats-score` | Read stored ATS analysis (no AI call) |
| GET | `/api/settings` | Fetch settings |
| PUT | `/api/settings` | Update settings |
| POST | `/api/parse-url` | Scrape job description from URL |
| POST | `/api/recommendations` | Get job recommendations (AI or Apify); body: `{ role?, location?, limit? }` |
| POST | `/api/recommendations/:recId/save` | Save a recommendation as a tracked job |
